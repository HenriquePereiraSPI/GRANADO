# Managing HTML Layout Editor

HTML Layout Editor Overview

At some point working with the Layout Editor might not be enough and the need for building more
sophisticated and complex screens may arise. For this purpose, the HTML Layout Editor
functionality can be used.

It should particularly be used when there is a need:

           For a more flexible way of designing the layout of User Controls on the screen
           To refresh some data on the screen asynchronously
           To build a custom user interface element

In other cases you should keep using Layout Editor.

  The HTML Layout Editor's functionality mainly allows for:
              Arranging the layout of current Process Builder Business Controls, User Inputs and
              Outputs, and Sub-DFCs with the use of HTML
                         For this purpose, a basic knowledge of HTML is required
              Calling an DFC asynchronously from the Web page
                         For this purpose, JavaScript and HTML knowledge is required
              Writing custom HTML-based User Interface elements
                         For this purpose, more advanced knowledge of HTML, JavaScript, and CSS is
                         required

 Conversion from Layout Editor

  To convert screens designed in the Layout Editor to the HTML Layout Editor, use the Convert to
  HTML Layout option from the HTML Layout Editor Right-Click Menu.

Using the HTML Layout Editor overrides the layout created with the use of the Layout Editor. To
go back to the Layout Editor, you need to use the Switch back to Layout Editor option from the
right-click menu.
