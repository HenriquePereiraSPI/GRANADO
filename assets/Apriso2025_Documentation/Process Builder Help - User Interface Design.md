# User Interface Design

When starting a new project, identify the number of Screen types you will need and validate the
Screen Layout of each of them. Ensure that they are all consistent. Once you agree with the
“standard” Layouts, then ensure that all the Screens are designed to use existing Layouts.

  It is important to keep this idea in mind: you must design the Screens for your Layout, not the
  Layouts for your Screens.

Design Rules

When a Screen header contains a Screen breadcrumb:

           Display the Screen description in the Screen breadcrumb, not its code
           The Header will also contain the Back and Exit buttons
           There is no need for a Back button in the footer Panel

When using the Grid Business Control:

           Display the View title in the grid header (there is no need for an additional Panel to display
           the Screen title)
           For all the Actions related to the data selected in the grid, define the Buttons directly in the
           grid (using the dedicated Template)
