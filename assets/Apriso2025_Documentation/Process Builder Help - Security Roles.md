# Security Roles

Each user that logs in to DELMIA Apriso and Process Builder can have specific security roles
assigned. The availability of certain features in Process Builder depends on what security roles are
assigned to the user who is logged in. The user may have more than one role assigned.

Role name           Permission

Process Functional  The user is permitted to:
Author
                               Edit processes, operations, DFCs, and their steps.
                               Edit screens, layouts, and views.
                               Run screens and DFCs in test mode.
                               Edit the user interface of existing elements.
                               Create projects, duplicate and edit project revisions.
                               Compare project revisions and merge them.

Process Links       This role is required to manage the business entities links displayed in
Configurator        the Links workspace.

Process Technical   The user is permitted to:
Author
                               Edit functions, inputs, and outputs.
                               Do whatever these roles are permitted to do: Process
                               Functional Author, Process Tester, Process User Interface
                               Author.
                               Create projects, duplicate and edit project revisions.
                               Compare project revisions and merge them.

Process Tester      The user is permitted to:

                               View processes, operations, DFCs, and their steps in read-only
                               mode.
                               Open screens, layouts, and views in read-only mode.
                               Run screens and DFCs in test mode.
                               Create projects, duplicate and view project revisions.
                               Compare project revisions.

Process User Interface The user is permitted to:

Author              View processes, operations, DFCs, and their steps in read-only

                    mode.

                    Open screens, layouts, and views in read-only mode.

                    Edit the user interface of steps.

                    Run screens and DFCs in test mode.

                    Create projects, duplicate and view project revisions.

                    Compare project revisions.

Issue Management    The user is permitted to:
Process Author
                               Access the Issue Flows Manager and manage fully Issue
                               Flows.
                               Access the Issue Properties and manage fully Issue Properties.

  You must be assigned the necessary roles to access M&M screens, which are available from the
  Managers menu in Process Builder. The Security tab of the FlexParts screen displays the
  existing roles assigned to the selected M&M screen.

For detailed information on security roles, refer to Role and Skill Configuration Help .
