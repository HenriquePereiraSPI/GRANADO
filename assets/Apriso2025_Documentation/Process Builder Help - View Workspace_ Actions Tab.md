# View Workspace_ Actions Tab

View Workspace: Actions Tab

The Actions tab in the View workspace lists all the Actions with their Properties defined for the
View. The Actions tab is available for all View Types.

You can see the configuration settings and also change the arrangement of Actions by using these
buttons:

Button Description
          Adds a new Action.
          Deletes the selected Action.
          Enables/disables the selected Action.

  This option can be enabled for Views that are in the active status. For details on how to
  enable it, see "Modifying Properties..." section of the Entity Status Types topic.

Moves the selected Action up/down on the list (available only if there is at least one Action
above/below the selected one). It is also possible to use drag-and-drop to change the
sequence of Actions.

         This option can be enabled for Views that are in the active status. For details on how to
         enable it, see "Modifying Properties..." section of the Entity Status Types topic.

The following table presents descriptions of the View settings. For additional details, see View
Action: General Tab.

Column             Description
Name
Enabled            If selected, the Action will be displayed in runtime.
Name               The name of the Action.
Type               The Action type determines what triggers the Action:

                              Button (Primary)
                              Calculated
                              Tab
                              Button (Secondary)
                              Numeric

   On Action       The name of the DFC or Action Script Function that will be executed when the
                   respective Action is triggered and before the target Screen is loaded. For more
   Portal          information, see On Action DFC and Action Scripts Overview.
   Action          The Action in the Portal.
   Open
   Popup           Opens the View in a Pop-up Window.
   View
   Next            The Action to be used as the Next Action.
   Action

You cannot edit any properties in the View workspace, as these are read-only fields.
