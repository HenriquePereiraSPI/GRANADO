# Screen Overview

A screen should be considered as a user interface. The composition of screens represents the
screen flows.

A screen is composed of one Layout and several Views. Views are linked to Layout Panels on the
screen or as pop-up windows through actions.

The navigation between screens, the interactions on the screen, and the running of business logic
is accomplished via actions.

These are some simple examples:

           Navigation between screens – the "Go To Screen" action invoked by a button.
           Interactions on a screen – switching tabs.
           Running business logic – a button that invokes a DFC.

This is the basic UI that should be described by a business flow. A screen contains displayed data
and controls to collect data, and it allows for Actions.
