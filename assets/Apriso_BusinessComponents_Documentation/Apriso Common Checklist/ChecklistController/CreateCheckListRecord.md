# CreateCheckListRecord

**Category:** Apriso/Common/Checklist
**Class:** `FlexNet.BusinessFacade.Common.ChecklistController`
**Assembly:** `FlexNet.BusinessFacade.Common.dll`
**Status:** Active

## Description

This Business Component method is used to create a new entry in the CHECK_LIST_HISTORY header table, which is used to save the history of the operator's answers to a particular checklist.

## Parameters

| IO | Name | Type | Required | Description |
|----|------|------|----------|-------------|
| Input | CheckListId | Integer | No | The ID of the checklist that the user is answering. Can be null if the user answers only one checkpoint. |
| Input | ResourceName | Char | No | The name of the Resource that the user is using. Can be null. |
| Input | ResourceType | Integer | No | The type of Resource that the user is using. Can be null. |
| Input | EmployeeId | Integer | Yes | The ID of the employee who is answering the checklist. Cannot be null. |
| Input | WipOrderNo | Char | No | The WIP Order number that the user is executing. Can be null. |
| Input | WipOrderType | Integer | No | The WIP Order type that the user is executing. Can be null. |
| Input | OprSequenceNo | Char | No | The WIP Operation Sequence number that the user is executing. Can be null. |
| Input | StartedOn | DateTime | Yes | The date/time that the user started answering questions. Cannot be null. |
| Output | CheckListHistoryID | Integer | No | The identifier of the current checklist history record generated by the system. |

## Description Parameters

| Name | Type | Description |
|------|------|-------------|
| StepSequenceNo | Integer | The WIP Operation Step Sequence number that the user is executing. |

## Validations

- If appropriate inputs are provided, the system validates if WIP Order, Operation or Step are valid. 
- CheckListId must exist in the CHECK_LIST table (if specified). 
- ResourceType and ResourceName must exist in the RESOURCE_ table (if specified). 
- EmployeeID must exist in the EMPLOYEE table. 
- StartedOn should have values different than DateTime.MinValue = 00:00:00.0000000, January 1, 0001.
