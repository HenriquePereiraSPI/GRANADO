# AddCheckListAnswer

**Category:** Apriso/Common/Checklist
**Class:** `FlexNet.BusinessFacade.Common.ChecklistController`
**Assembly:** `FlexNet.BusinessFacade.Common.dll`
**Status:** Active

## Description

This Business Component method is used to create new data with the operator's answers.

## Parameters

| IO | Name | Type | Required | Description |
|----|------|------|----------|-------------|
| Input | CheckListHistoryID | Integer | Yes | The identifier of the current checklist history record generated by the system. Cannot be null. |
| Input | CheckPointID | Integer | Yes | The ID of the checkpoint that was used to show the question. Cannot be null. |
| Input | CheckPointValueId | Integer | No | The ID of an answer the user gave. Can be null. |
| Input | Value | Char | No | The value of an answer the user gave. Can be null. |
| Input | AnsweredOn | DateTime | No | The date/time the user answered the question. |
| Input | EmployeeID | Integer | No | The ID of the employee. |

## Validations

- CheckListHistoryID must exist in the CHECK_LIST_HISTORY table. 
- CheckPointID must exist in the CHECK_POINT table. 
- CheckPointValueId must exist in the CHECK_POINT_VALUE table (if specified). 
- If AnsweredOn was provided it, should have values different than DateTime.MinValue = 00:00:00.0000000, January 1, 0001.

## System Processing

The system adds a checkpoint history record and fills additional fields if specified.
