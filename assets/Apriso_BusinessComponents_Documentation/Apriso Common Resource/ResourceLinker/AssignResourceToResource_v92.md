# AssignResourceToResource_v92

**Category:** Apriso/Common/Resource
**Class:** `FlexNet.BusinessFacade.Resources.ResourceLinker`
**Assembly:** `FlexNet.BusinessFacade.Resources.dll`
**Status:** Active

## Description

This Business Component method is used to assign a Resource to a Resource. The BC can have the following purposes:
 
 
- It can limit the user Task List to specific tasks linked to Rresources. 
- It can assign one or more Equipments (Resources) to an Employee (a Resource) from a specific date and for a specific duration in minutes. 
- It can assign a sub-Resource (tool) to a Resource (machine). 
 

The link between the two Resources is a record in the RESOURCE_RESOURCE_LINK table. If a ResourceLinkClassID is inputted, the BC will persist the class of the sub-Resource in the RESOURCE_RESOURCE_LINK table.

## Parameters

| IO | Name | Type | Required | Description |
|----|------|------|----------|-------------|
| Input | ParentResourceName | Char | Yes | The Resource name to assign a Resource to. |
| Input | ParentResourceType | Integer | Yes | The Resource type to assign a Resource to. |
| Input | ResourceName | Char | No | The Resource name to assign. |
| Input | ResourceType | Integer | No | The Resource type to assign. |
| Input | ResourceLinkClassID | Integer | No | The Resource link class. |
| Input | AssignedOn | DateTime | No | The date the assignment took place. This is optional and is in UTC. |
| Input | Duration | Integer | Yes | The duration in seconds. |

## Validations

- The system validates that the Resource link class exists if ResourceLinkClassID is specified (the RESOURCE_LINK_CLASS table). 
- The system determines that the ParentResourceName and ParentResourceType Inputs have been provided. If not, the system returns an error message. 
- The system validates that the parent Resource exists. If not, it returns an error message. 
- The system validates that ResourceType and ResourceName exist. If not, the system returns an error message.

## System Processing

- The system creates a record in the RESOURCE_RESOURCE_LINK table with the inputted values. 
- If the Duration parameter is specified, the system persists the ReleasedOn field based on AssignedOn and Duration (this parameter is specified in seconds).

## Pre-Conditions

The Resource and parent Resource must exist. The Resource link class must exist (if ResourceLinkClassID is specified).
