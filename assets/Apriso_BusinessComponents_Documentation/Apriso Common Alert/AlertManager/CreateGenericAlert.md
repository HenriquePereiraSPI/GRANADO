# CreateGenericAlert

**Category:** Apriso/Common/Alert
**Class:** `FlexNet.BusinessFacade.Alerts.AlertManager`
**Assembly:** `FlexNet.BusinessFacade.Alerts.dll`
**Status:** Deprecated

## Description

The purpose of this method is to publish an alert and keep the alert information in the database.

## Parameters

| IO | Name | Type | Required | Description |
|----|------|------|----------|-------------|
| Input | AlertTypeID | Integer | Yes | The type of alert generated. |
| Input | GeneratedOn | DateTime | Yes | Date and time when the alert event was generated by the source. This is not the date time when the alert was created in this table. |
| Input | MessageID | Char | No | Uniquely identifies the type of error or alert generated throughout the DELMIA Apriso system. |
| Input | Message | Char | No | The text message for the alarm. This may be a standard error message in DELMIA Apriso. |
| Input | CreateOperationID | Integer | No | The DFC in DELMIA Apriso which controls the creation of the alert. |
| Input | AlertClassID | Integer | Yes | An alert class assigned and controlled by System administrator and/or by DELMIA Apriso. Valid classes are in the ALERT_CLASS table and are defaulted Alert Class 1 through 9. |
| Input | ParentAlertID | Integer | No | The ID of the Alert that generated this Alert, if defined. |
| Input | DispositionTestID | Integer | No | Id of the disposition test. |
| Input | Characteristic | Char | No | Id of the characteristic. |
| Input | TestAttribute | Char | No | Attribute of the test. |
| Input | Value | Char | Yes | Value of the test. |
| Input | WipOrderNo | Char | No | Number of the wip order. |
| Input | WipOrderType | Integer | No | Type of the wip order. |
| Input | OprSequenceNo | Char | No | The operation sequence number. |
| Input | SourceBCMethodID | Integer | No | The id of Business Component method. |
| Input | SourceTaskID | Integer | No | The id of source task. |
| Input | ExternalReference | Char | No | Link to SPC Chart Group. |
| Output | AlertID | Integer | No | The ID of the Alert. |

## Validations

System performs the following validations:
 
 
- AlertTypeID passed is valid and active in ALERT_TYPE. 
- AlertClassID passed is valid and active in ALERT_CLASS. 
- WipOrderNo, WipOrderType and OprSequenceNo are valid and active in the WIP_ORDER and WIP_OPERATION tables or are null. 
- ParentAlertID is valid and active in the ALERT table or is 0. 
- DispositonTestID is valid and active in the DISPOSITION_TEST table or is 0. 
- Characteristic is a valid Characteristic and active in the CHARACTERISTIC table or is null. 
- TestAttribute is a valid Attribute and active in the ATTRIBUTE table or is null. 
- SourceBCMethodID is a valid BC Method in the BUSINESS_COMPONENT_METHOD table or is 0. 
- SourceTaskID is a valid ID in the TASK table or is 0. 
- Priority is in the PRIORITY table 
- ResponseOperationID is a valid ID in the DFC_REVISION table. 
- ResourceName, ResourceTypeID is a valid in the RESOURCE table or is "" and 0.

## System Processing

During processing, system:
 
 
- Performs validations as stated above, 
- Computes the GeneratedOn and ExpirationDate date/time, 
- Keeps one ALERT record and one ALERT_DETAIL record.
