# CreateGenericAlert_v3

**Category:** Apriso/Common/Alert
**Class:** `FlexNet.BusinessFacade.Alerts.AlertManager`
**Assembly:** `FlexNet.BusinessFacade.Alerts.dll`
**Status:** Deprecated

## Description

The purpose of this method is to publish an alert and keep the alert information in the database.

## Validations

The system performs the following validations:
 
 
- AlertTypeID passed is valid and active in ALERT_TYPE. 
- AlertClassID passed is valid and active in ALERT_CLASS. 
- WipOrderNo, WipOrderType and OprSequenceNo are valid and active in the WIP_ORDER and WIP_OPERATION tables or are null. 
- ParentAlertID is valid and active in the ALERT table or is 0. 
- DispositonTestID is valid and active in the DISPOSITION_TEST table or is 0. 
-  

Characteristic **** is valid in the CHARACTERISTIC table 

when DispositonTestID is not provided. TestAttribute, Characteristic are valid in the ATTRIBUTE table when Characteristic and TestAttribute are provided and CharacteristicType equals Attribute.
 
  
 
 
- SourceBCMethodFUID is a valid BC Method in the BUSINESS_COMPONENT_METHOD table. 
- If SourceBCMethodFUID not found, SourceBCMethodID is a valid BC Method in the BUSINESS_COMPONENT_METHOD table. If SourceBCMethodID is 0 then NULL value is set. 
- SourceTaskID is a valid ID in the TASK table or is 0. 
- PriorityID is in the PRIORITY table. 
- ResponseOperationID is a valid ID in the DFC_REVISION table. 
- ResourceName and ResourceType are valid in the RESOURCE table or are "" and 0 respectively. 
- Facility is a valid in FACILITY table or is "". 
-  

WorkCenter is valid in WORK_CENTER or is "".
  
-  

Container is valid in CONTAINER or is "".
  
-  

ProductID is valid in PRODUCT or is <= 0.
  
-  

LotNo is valid in LOT (if ProductID > 0) or is "".

## System Processing

During processing, the system:
 
 
- Performs validations as stated above. 
- Computes the GeneratedOn and ExpirationDate date/time. 
- Keeps one ALERT record and one ALERT_DETAIL record. 
- If DispositionTestID input is specified following columns are set in ALERT_DETAIL table based on DISPOSITION_TEST: UomCode, LowerControlLimit, UpperControlLimit, ConformingToControl, LowerSpecyficationLimit, UpperSpecificationLimit, ConformingToSpecification, Characteristic, Value, TargetValue, TestAttribute. 
- If DispositionTestID input is not specified ALERT_DETAIL Value, TestAttribute and Characteristic are set based on correcponding BC inputs.
