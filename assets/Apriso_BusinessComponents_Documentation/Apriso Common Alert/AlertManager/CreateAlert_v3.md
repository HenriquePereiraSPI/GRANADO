# CreateAlert_v3

**Category:** Apriso/Common/Alert
**Class:** `FlexNet.BusinessFacade.Alerts.AlertManager`
**Assembly:** `FlexNet.BusinessFacade.Alerts.dll`
**Status:** Deprecated
**Keywords:** Alert, CreateAlert

## Description

Publishes an alert and keeps the alert information in the database. Typical ways to invoke this BC method are as follows:
 
 
- An event generated by: 
 
- Machine Integrator 
- An external source 
- A business component in a DFC. 
 
- A DFC processes these alert flows in the background. This DFC will be designed to retrieve and determine the required information that will be processed by the CreateAlert_v3 BC method. 
- The CreateAlert_v3 BC method is invoked from a DFC.

## Validations

- Validation passes if AlertTypeID is valid and active in ALERT_TYPE. 
- Validation passes if AlertClassID is valid and active in ALERT_CLASS. 
- WipOrderNo, WipOrderType, OprSequenceNo and StepSequenceNo are valid and active in the WIP_ORDER , WIP_OPERATION and WIP_OPERATION_STEP tables. Validation fails if they are null. 
- ParentAlertID is valid and active in the ALERT table. Validation fails if it is 0. 
- DispositonTestID is valid and active in the DISPOSITION_TEST table. Validation fails if it is 0. 
- Validation passes when DispositionTestID is not provided and: 
 
- Characteristic is valid in the CHARACTERISTIC table 
- TestAttribute and Characteristic are valid in the ATTRIBUTE table when Characteristic and TestAttribute are provided and CharacteristicType equals Attribute. 
 
- SourceBCMethodFUID is a valid BC Method in the BUSINESS_COMPONENT_METHOD table. 
- If SourceBCMethodFUID not found, SourceBCMethodID is a valid BC Method in the BUSINESS_COMPONENT_METHOD table. If SourceBCMethodID is 0 then NULL value is set. 
- SourceTaskID is a valid ID in the TASK table. Validation fails if it is 0. 
- Validation passes if Priority is in the PRIORITY table as determined by FLX_ALERT_PRIORITY_DETERMINATION. 
- Validation passes if ResponseOperationID determined by FLX_ALERT_RESPONSE_DETERMINATION is valid ID in DFC_REVISION table. 
- Validation passes if ResourceClassID is a valid ID in the RESOURCE_CLASS table or is 0 as determined by FLX_ALERT_RECIPIENT_DETERMINATION. 
- Validation passes if ResourceTypeID is a valid ID in the RESOURCE_TYPE table or is 0, as determined by FLX_ALERT_RECIPIENT_DETERMINATION. 
- Validation passes if EmployeeID is a valid ID in the EMPLOYEE table or is 0, as determined by FLX_ALERT_RECIPIENT_DETERMINATION. 
- Validation passes if RoleID is a valid ID in the ROLE table or is 0, as determined by FLX_ALERT_RECIPIENT_DETERMINATION. 
- Validation passes if CommunicationType is a valid CommunicationType in the COMMUNICATION_TYPE table or is 0, as determined by FLX_ALERT_RECIPIENT_DETERMINATION.

## System Processing

- Performs validations as stated above. 
- Invokes the following Determination Functions: 
- FLX_ALERT_PRIORITY_DETERMINATION 
- FLX_ALERT_RESPONSE_DETERMINATION 
- FLX_ALERT_RECIPIENT_DETERMINATION 
- Performs validations against data returned by Determination. 
- Computes the GeneratedOn and ExpirationDate date/time. 
- Retrieves the Communications Type and Communications Value as well as invokes the communications: 
 
- Communications type set as Homepage indicates that an Alert should be displayed to the user, 
- Other Communications types are implemented on as add required basis. 
 
- Keeps one ALERT record and one ALERT_DETAIL record. 
- Keeps one or many ALERT_RECIPIENT records as defined by FLX_ALERT_RECIPIENT_DETERMINATION. 
- If DispositionTestID is specified, the following columns are set in ALERT_DETAIL table based on DISPOSITION_TEST: UomCode, LowerControlLimit, UpperControlLimit, ConformingToControl, LowerSpecificationLimit, UpperSpecificationLimit, ConformingToSpecification, Characteristic, Value, TargetValue, TestAttribute. 
- If DispositionTestID is not specified, then ALERT_DETAIL Value, TargetValue, TestAttribute and Characteristic are set based on corresponding BC inputs. If Characteristic is specified and CharacteristicType equals "Variable", then UpperSpecificationLimit, UpperControlLimit, LowerControlLimit, LowerSpecificationLimit, TargetValue (overrides input) and UomCode are set based on CHARACTERISTIC.
