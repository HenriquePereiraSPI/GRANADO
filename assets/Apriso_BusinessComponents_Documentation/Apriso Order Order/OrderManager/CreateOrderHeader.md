# CreateOrderHeader

**Category:** Apriso/Order/Order
**Class:** `FlexNet.BusinessFacade.Common.OrderManager`
**Assembly:** `FlexNet.BusinessFacade.Common.dll`
**Status:** Active

## Description

The purpose of this Business Component is to create an order header. It generates a record in the ORDER_HEADER table. This Business Component also allows the user to link the created order header to an existing WipOrder. This is done by inputting the WipOrderNo and WipOrderType of the Wip Order. The created Order Header is the output of this Business Component.

## Validations

- Order Type, Order Status and Priority are mandatory inputs 
- If OrderNo is inputted, the system checks that the inputted OrderNo is unique for the inputted OrderType. If not, system displays an error message.
 **Note:** The inputs of type Date are not mandatory inputs. However, a date must be entered (PB constraint for date-type fields). If any of the dates are not required, a date smaller than 1st January 1905 or greater than the 31st December 2135 should be inputted (will be processed as no date). 
- If TextLong is inputted, system checks that LanguageID is inputted as well. If not, system displays an error message. 
- If a WipOrder is inputted, system checks that this WipOrder exists in the DB. If not, system displays an error message. 
- All data must be consistent with DB FK constraints.

## System Processing

- System creates a record in the ORDER_HEADER table. 
- If OrderNo is not inputted, then a new order number will be retrieved from the sequence provider for the ORDERNO sequence. This is a system-based sequence, not facility-based. 
- If TextLong is inputted, system generates a record in the TEXT_TRANSLATION table. 
- If OrderDate is inputted, system generates a record in the ORDER_DATE table. 
- The (created) OrderNo is the output of the Business Component.
