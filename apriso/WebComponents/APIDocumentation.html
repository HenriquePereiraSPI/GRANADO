Referência de APIs

Catálogo das APIs de servidor do Granado expostas no Apriso (prefixo GRD_API_). Cada API recebe um payload JSON e retorna um envelope padrão com Success, Message e Data. A documentação abaixo descreve, por API, os campos de entrada, os de saída e exemplos prontos para teste.

GRD_API_AppointChecklist

Registra o apontamento de um checklist para uma ordem (WIP Order) em um recurso específico. Recebe a lista de checkpoints respondidos — cada um com seu valor (1 = conforme / 0 = não conforme) — e devolve o ID do apontamento gerado. Use ao concluir um checklist de turno, de POP ou de inspeção em processo.

Chamada

API GRD_API_AppointChecklist

Operação de servidor invocada com um corpo JSON. O retorno é sempre o envelope padrão (Success / Message / Data).

Inputs

Corpo da requisição (JSON):

{
  "CheckListID": "100000002",
  "WIPOrderNo": "1000000001",
  "ResourceName": "100000032",
  "CheckPoints": [
    { "CheckpointID": "100000002", "CheckpointValue": { "Checked": 1, "CustomValue": "1.500", "AditionalInformation": "Alguma informação do checklist" } },
    { "CheckpointID": "100000003", "CheckpointValue": { "Checked": 1, "CustomValue": "", "AditionalInformation": "" } },
    { "CheckpointID": "100000004", "CheckpointValue": { "Checked": 0, "CustomValue": "", "AditionalInformation": "" } },
    { "CheckpointID": "100000005", "CheckpointValue": { "Checked": 1, "CustomValue": "", "AditionalInformation": "" } }
  ]
}

Campos de entrada

Campo Tipo Obrigatório Descrição
CheckListID string Sim ID do checklist que está sendo apontado.
WIPOrderNo string Não Número da ordem (WIP Order) à qual o apontamento se vincula. Opcional — quando não for informar, envie "-1".
ResourceName string Não Identificador do recurso/equipamento onde o checklist foi executado. Opcional — quando não for informar, envie "-1".
CheckPoints array<object> Sim Lista dos checkpoints respondidos. Cada item contém os campos abaixo.
CheckpointID string Sim ID do checkpoint dentro do checklist.
CheckpointValue object Sim Valor apontado para o checkpoint. Contém os campos abaixo.
Checked number (0 | 1) Sim Estado do checkpoint: 1 = conforme / sim, 0 = não conforme / não.
CustomValue string Não Valor customizado do checkpoint (ex.: medição "1.500"). Envie "" quando não houver valor.
AditionalInformation string Não Observação/texto livre associado ao apontamento do checkpoint. Envie "" quando não houver.

Outputs

Envelope de resposta (JSON):

{
  "Success": true,
  "Message": "Checkpoint apontamdo com sucesso.",
  "Data": "100000015"
}

Campos de saída

Campo Tipo Descrição
Success boolean Indica se o apontamento foi concluído com sucesso.
Message string Mensagem de retorno — confirmação em caso de sucesso ou detalhe do erro.
Data string ID do registro de apontamento gerado.

Notas

  • Envelope padrão: todas as APIs GRD_API_ retornam Success, Message e Data. Verifique Success antes de usar Data.
  • Opcionais: WIPOrderNo e ResourceName são opcionais — quando não houver valor a informar, envie "-1" no lugar.
  • CheckPoints: envie um item para cada checkpoint do checklist; itens ausentes não são apontados.
  • Valores: dentro de CheckpointValue, Checked aceita apenas 0 ou 1; CustomValue e AditionalInformation são opcionais (envie "" quando não houver).

GRD_API_VerifyCheckList

Verifica se o checklist já foi preenchido no período para um ou mais recursos. Recebe uma lista de verificações (recurso + checklist + período) e devolve, para cada item, se já está preenchido (Done).

Chamada

API GRD_API_VerifyCheckList

Operação de servidor invocada com um corpo JSON. Diferente das demais, o corpo é um array e a resposta também é um array (um item de retorno por item enviado).

Inputs

Corpo da requisição (JSON) — lista de verificações:

[
  {
    "ResourceName": "BALANCA_A",
    "CheckListCode": "CHECKLIST_BALANCA_A",
    "Period": "DAY | WEEK | MONTH"
  },
  {
    "ResourceName": "BALANCA_B",
    "CheckListCode": "CHECKLIST_BALANCA_B",
    "Period": "DAY | WEEK | MONTH"
  }
]

Campos de cada item de entrada

Campo Tipo Obrigatório Descrição
ResourceName string Sim Nome do recurso (equipamento) a verificar.
CheckListCode string Sim Código do checklist a verificar.
Period string Sim Janela de verificação: DAY (dia) · WEEK (semana) · MONTH (mês).

Outputs

Resposta (JSON) — lista com o resultado de cada verificação:

[
  {
    "ResourceName": "BALANCA_A",
    "CheckListCode": "CHECKLIST_BALANCA_A",
    "Done": 0
  },
  {
    "ResourceName": "BALANCA_B",
    "CheckListCode": "CHECKLIST_BALANCA_B",
    "Done": 1
  }
]

Campos de cada item de saída

CampoTipoDescrição
ResourceNamestringRecurso verificado (eco do input).
CheckListCodestringChecklist verificado (eco do input).
Donenumber1 = checklist já preenchido no período · 0 = ainda não preenchido.

Notas

  • Entrada e saída em lote: a API recebe e retorna um array; a resposta traz um item para cada verificação enviada.
  • Período: Period define a janela em que o preenchimento é considerado válido (dia, semana ou mês corrente).
  • Sem envelope Success/Message: diferente das demais APIs, o retorno é diretamente a lista de resultados.

Histórico com % de preenchimento

Lista o histórico de apontamentos de checklist (somente o cabeçalho) de uma ordem, com totais e o percentual de checkpoints marcados (Checked = 1) em cada apontamento. Usa uma CTE que agrega CHECK_POINT_HISTORY por CheckListHistoryID, extraindo $.Checked do JSON com JSON_VALUE.

Query

;WITH cteCheckPointHistory AS
(
  SELECT
    CheckListHistoryID,
    COUNT(*) AS TotalChecks,
    SUM(CAST(JSON_VALUE(Value_, '$.Checked') AS INT)) AS SumChecked,
    CAST(SUM(CAST(JSON_VALUE(Value_, '$.Checked') AS INT)) * 100.0 / COUNT(*) AS DECIMAL(5,2)) AS PercentOfTotalChecked
  FROM CHECK_POINT_HISTORY
  GROUP BY CheckListHistoryID
)
SELECT
  CheckListHistory.ID AS CheckListHistoryID,
  CheckListBase.CheckListCode,
  CheckListHistory.ResourceName,
  CheckListHistory.WipOrderNo,
  CheckListHistory.CreatedBy,
  CheckListHistory.CreatedOn,
  cte.*
FROM CHECK_LIST_HISTORY CheckListHistory
INNER JOIN cteCheckPointHistory cte ON CheckListHistory.ID = cte.CheckListHistoryID
INNER JOIN CHECK_LIST CheckListBase ON CheckListHistory.CheckListID = CheckListBase.ID
WHERE CheckListHistory.WipOrderNo = '1000000001'
ORDER BY CheckListHistory.CreatedOn

Parâmetro

Filtro Descrição
CheckListHistory.WipOrderNo Número da ordem (WIP Order) cujos apontamentos de checklist se deseja listar.

Colunas retornadas

Coluna Descrição
CheckListHistoryID ID do apontamento de checklist (CHECK_LIST_HISTORY.ID).
CheckListCode Código do checklist apontado.
ResourceName Recurso/equipamento onde o checklist foi apontado.
WipOrderNo Número da ordem vinculada ao apontamento.
CreatedBy Usuário que registrou o apontamento.
CreatedOn Data/hora do apontamento (usada na ordenação).
TotalChecks Total de checkpoints do apontamento.
SumChecked Quantidade de checkpoints marcados (Checked = 1).
PercentOfTotalChecked Percentual de preenchimento: SumChecked × 100 / TotalChecks (DECIMAL 5,2).

Notas

  • Header apenas: traz uma linha por apontamento (CHECK_LIST_HISTORY) — para o detalhe de cada resposta use a query Detalhe das respostas.
  • JSON: Value_ guarda o valor do checkpoint em JSON; JSON_VALUE(Value_, '$.Checked') extrai o flag 0/1 para agregação.
  • Percentual: PercentOfTotalChecked usa 100.0 (decimal) para evitar divisão inteira e arredonda em DECIMAL(5,2).

Detalhe das respostas

Retorna o detalhe de cada checkpoint respondido de um apontamento específico, com a descrição traduzida do checkpoint e os campos do valor (Checked, CustomValue, AditionalInformation) extraídos do JSON em CHECK_POINT_HISTORY.Value_.

Query

SELECT
  CheckListHistory.ID AS CheckListHistoryID,
  CheckListHistory.ResourceName,
  CheckListHistory.WipOrderNo,
  CheckListHistory.CreatedBy,
  CheckListHistory.CreatedOn,
  dbo.fnGetTranslation(CheckPoints.TextID, 1033) AS CheckPointDescription,
  JSON_VALUE(CheckPointHistory.Value_, '$.Checked')   AS Checked,
  JSON_VALUE(CheckPointHistory.Value_, '$.CustomValue')    AS CustomValue,
  JSON_VALUE(CheckPointHistory.Value_, '$.AditionalInformation')  AS AditionalInformation
FROM CHECK_LIST_HISTORY CheckListHistory
INNER JOIN CHECK_POINT_HISTORY CheckPointHistory ON CheckListHistory.ID = CheckPointHistory.CheckListHistoryID
LEFT JOIN CHECK_POINT CheckPoints ON CheckPointHistory.CheckPointID = CheckPoints.ID
WHERE CheckPointHistory.CheckListHistoryID = '100000004'

Parâmetro

Filtro Descrição
CheckPointHistory.CheckListHistoryID ID do apontamento de checklist (CHECK_LIST_HISTORY.ID) — o mesmo CheckListHistoryID retornado pela query de histórico.

Colunas retornadas

Coluna Descrição
CheckListHistoryID ID do apontamento de checklist.
ResourceName Recurso/equipamento do apontamento.
WipOrderNo Número da ordem vinculada.
CreatedBy Usuário que registrou o apontamento.
CreatedOn Data/hora do apontamento.
CheckPointDescription Descrição traduzida do checkpoint via dbo.fnGetTranslation (LCID 1033).
Checked Flag 0/1 do checkpoint, extraído de $.Checked.
CustomValue Valor customizado, extraído de $.CustomValue (pode vir vazio).
AditionalInformation Observação livre, extraída de $.AditionalInformation (pode vir vazia).

Notas

  • Detalhe por linha: uma linha por checkpoint respondido do apontamento informado.
  • LEFT JOIN: o join com CHECK_POINT é LEFT para não perder respostas cujo checkpoint base tenha sido removido — nesses casos CheckPointDescription vem NULL.
  • JSON: os três últimos campos saem de CHECK_POINT_HISTORY.Value_ (o mesmo objeto enviado em CheckpointValue no apontamento via API).
