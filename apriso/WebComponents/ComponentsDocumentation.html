<granado-simple-card>

Card minimalista com faixa de destaque colorida, título e subtítulo. Indicado para listar itens (ordens de produção, status de linha, lotes) onde a cor transmite o contexto de forma rápida. A faixa pode ir em qualquer lado (border-position), aceita um icon opcional, e título/ícone/borda derivam de color (com overrides por text-color, icon-color, border-color). Tamanho e cor de título e subtítulo também podem ser ajustados via title-size, subtitle-size e subtitle-color. Pode ser tornado clicável passando onclickevent.

Import

<!-- Inclua o script no <head> (padrão Apriso) -->
<script src="[AprisoScripts]/WebComponents/granado-simple-card.js"></script>

Uso básico

Apenas title e subtitle. A faixa lateral assume a cor padrão (verde Granado #1C5C31). 

<granado-simple-card
    title="OP-2025-0042"
    subtitle="Shampoo Phebo · 5000un">
</granado-simple-card>

Cores customizadas

O atributo color aceita qualquer cor CSS válida (hex, rgb, hsl, nome). Útil para sinalizar contexto: vermelho para alertas, ouro para avisos, azul para informativos. 

<granado-simple-card
    title="Linha 03 parada"
    subtitle="Aguardando troca de formato"
    color="#8C1A1A">
</granado-simple-card>

<granado-simple-card
    title="Lote aprovado"
    subtitle="Liberado para envase"
    color="#9A7B1C">
</granado-simple-card>

<granado-simple-card
    title="Aviso geral"
    subtitle="Manutenção programada às 22h"
    color="#1A4A8C">
</granado-simple-card>

Interativo (clicável)

Quando onclickevent é definido, o card ganha cursor pointer, sombra ampliada no hover e um leve lift de 1px. O conteúdo do atributo é executado como JS no contexto do elemento. 

<granado-simple-card
    title="Clique aqui"
    subtitle="Card interativo com hover"
    color="#1A4A8C"
    onclickevent="alert('Card simples clicado')">
</granado-simple-card>

Posição da faixa, ícone e cores

A faixa de destaque pode ir em qualquer lado via border-position (left · top · right · bottom). Um icon opcional aparece num badge circular no topo-direito. Por padrão título, ícone e borda fina derivam de color; cada um pode ser sobrescrito por text-color, icon-color e border-color. Use border-color="transparent" para remover a borda fina. 

<!-- Faixa no topo + ícone; tudo derivado de color -->
<granado-simple-card
    title="Em Pesagem"
    subtitle="OP-2025-0416 · Loção Hidratante"
    color="#1A4A8C"
    border-position="top"
    icon="🔄">
</granado-simple-card>

<!-- Sobrescrevendo cada cor + sem borda fina -->
<granado-simple-card
    title="Cor custom"
    color="#8C1A1A"
    text-color="#103E20"
    icon-color="#9A7B1C"
    border-color="transparent"
    icon="🎨">
</granado-simple-card>

Tamanho e cor do título/subtítulo

O tamanho da fonte do título e do subtítulo pode ser ajustado com title-size (default 16px) e subtitle-size (default 13px) — ambos aceitam qualquer unidade CSS (px, rem, em). A cor do subtítulo é controlada por subtitle-color (default #4B5563); a cor do título continua em text-color. 

<granado-simple-card
    title="Tudo custom"
    subtitle="size e cor de ambos"
    color="#8C1A1A"
    text-color="#8C1A1A"
    title-size="20px"
    subtitle-color="#8C1A1A"
    subtitle-size="15px">
</granado-simple-card>

Estilo & customização

Estilos do container (largura, margem, flex) são aplicados diretamente via atributo style no elemento — por exemplo <granado-simple-card style="flex:1;min-width:220px">. O componente usa Light DOM, então é possível inspecionar e estender o markup via DevTools se for necessário.

<granado-complete-card>

Card centrado com ícone no topo, título, subtítulo e badge de status — visual inspirado nos botões de seleção de sala da tela de pesagem. Por padrão a color pinta a borda, o ícone, o título e o badge, dando leitura imediata de contexto; cada elemento pode ser sobrescrito (text-color, icon-color, border-color, status-color). Útil para grids de seleção (salas, balanças, recursos, status de linha).

Import

<!-- Inclua o script no <head> (padrão Apriso) -->
<script src="[AprisoScripts]/WebComponents/granado-complete-card.js"></script>

Uso básico

Combinação típica de icon + title + subtitle + status. Sem color, o card usa o verde Granado padrão (#1C5C31). 

<granado-complete-card
    icon="A"
    title="Sala A"
    subtitle="4 Balanças"
    status="Disponível">
</granado-complete-card>

Cores customizadas

O atributo color aceita qualquer cor CSS válida e é aplicado tanto à borda quanto ao fundo do badge. Use um grid simples para listas de salas, recursos ou áreas com status diferentes. 

<granado-complete-card icon="A" title="Sala A" subtitle="4 Balanças"
    status="Disponível"></granado-complete-card>

<granado-complete-card icon="B" title="Sala B" subtitle="3 Balanças"
    color="#9A7B1C" status="1 em uso"></granado-complete-card>

<granado-complete-card icon="C" title="Sala C" subtitle="2 Balanças"
    color="#8C1A1A" status="Manutenção"></granado-complete-card>

<granado-complete-card icon="D" title="Sala D" subtitle="6 Balanças"
    color="#1A4A8C" status="Reservada"></granado-complete-card>

Atributos opcionais

Todos os atributos são opcionais — o card renderiza apenas o que receber. Omitir status esconde o badge; omitir icon remove a área superior. O icon aceita emoji ou texto curto (sigla, OP, código). Bom para variações compactas. 

<!-- Apenas icon + title + subtitle (sem status) -->
<granado-complete-card
    icon="⚙️"
    title="Configurações"
    subtitle="Sistema · permissões · usuários">
</granado-complete-card>

<!-- Sem icon, sem status -->
<granado-complete-card
    title="Apenas título"
    subtitle="Sem ícone, sem badge"
    color="#9A7B1C">
</granado-complete-card>

Interativo (clicável)

Quando onclickevent é definido, o card vira clicável: cursor pointer, sombra ampliada no hover e lift de 1px. Ideal para grids de seleção (selecionar sala, balança, recurso). 

<granado-complete-card
    icon="D"
    title="Sala D"
    subtitle="6 Balanças"
    color="#1A4A8C"
    status="Reservada"
    onclickevent="alert('Sala D selecionada')">
</granado-complete-card>

Cores por elemento

Por padrão a borda, o ícone, o título e o badge de status derivam de color. Cada um pode ser sobrescrito individualmente com text-color (título), icon-color (ícone), border-color (borda) e status-color (badge). Use border-color="transparent" para o card sem borda. 

<!-- Tudo derivado de color -->
<granado-complete-card icon="⚙️" title="Tudo na cor"
    color="#1A4A8C" status="Padrão"></granado-complete-card>

<!-- Sobrescrevendo cada elemento -->
<granado-complete-card icon="🎨" title="Cores custom"
    color="#1A4A8C"
    text-color="#103E20"
    icon-color="#9A7B1C"
    status-color="#1C5C31"
    status="Misto"></granado-complete-card>

<!-- Card sem borda -->
<granado-complete-card icon="✓" title="Sem borda"
    color="#1C5C31" border-color="transparent" status="Aprovado"></granado-complete-card>

Estilo & customização

Largura, margem e flex são aplicados via atributo style direto no elemento — por exemplo <granado-complete-card style="flex:1;min-width:160px">. Para grids uniformes, combine display:flex; gap:12px no container pai com flex:1; min-width:Xpx em cada card.

<granado-empty-card>

Container vazio com a mesma moldura do <granado-simple-card> (faixa lateral, fundo, borda, sombra), porém sem conteúdo pré-definido. O que for colocado dentro do elemento — KPIs, tabelas, formulários, gráficos — fica enquadrado pela casca. Use quando precisar do visual de card mas o conteúdo é arbitrário.

Internamente o componente usa um MutationObserver para reincorporar filhos inseridos depois do connectedCallback (parser ou JS), então o encaixe funciona tanto em HTML estático quanto em conteúdo adicionado dinamicamente.

Com title definido, o card mostra um cabeçalho eyebrow em uppercase (9px, peso 900, espaçamento .2em) acima do conteúdo.

Sem título

Quando title é omitido, o eyebrow não é renderizado e o conteúdo ocupa toda a área útil — útil para enquadrar listas, tabelas ou formulários sem cabeçalho extra.

Qualquer conteúdo HTML pode ser colocado aqui. O card serve apenas como moldura — parágrafos, formulários, gráficos, listas, tudo é renderizado como está dentro do slot interno. 

<granado-empty-card>
    <p>Qualquer conteúdo HTML pode ser colocado aqui.</p>
</granado-empty-card>

API

Atributos

Nome Tipo Default Descrição

Comportamento

Filhos do elemento são reposicionados em um slot interno (.gec-slot) que vive dentro da casca .gec-card. Um MutationObserver escuta inserções tardias (parser do navegador ou JS dinâmico) e reencaixa o conteúdo automaticamente — sem necessidade de chamar render() manualmente.

Estilo & customização

Largura, margem e flex ficam no atributo style do próprio elemento. O conteúdo interno fica livre de estilizações impostas pelo card — qualquer CSS aplicado aos filhos (incluindo style inline ou classes externas) continua funcionando dentro do slot.

<granado-table>

Tabela de dados com card embutido (mesma moldura visual do <granado-simple-card>: faixa lateral colorida, fundo, borda, sombra). Suporta paginação, filtros por coluna, filtro global, ordenação, renderização customizada de células, eventos de single/double click e exportação para CSV e PDF — tudo configurável via atributos.

Os dados (columns, rows) e os handlers JS (onrender, onclickevent) são strings JSON ou JS no atributo, então tipicamente o setup é feito via setAttribute para escapar e serializar limpo.

Import

<!-- Inclua o script no <head> (padrão Apriso) -->
<script src="[AprisoScripts]/WebComponents/granado-table.js"></script>

Uso básico

Defina columns (array de objetos) e rows (array de objetos) — as keys das colunas devem bater com as keys dos objetos de linha. Setup via JS evita a dor de cabeça de escapar JSON inline no HTML. 

<granado-table id="tbl"></granado-table>

<script>
  // Defina as colunas (key + label, no mínimo)
  const columns = [
    { key: 'cod',    label: 'Código',     mono: true, width: '100px' },
    { key: 'mat',    label: 'Material',   width: '180px' },
    { key: 'pesado', label: 'Pesado (kg)', mono: true, align: 'right' },
    { key: 'status', label: 'Status' }
  ];

  // Cada linha é um objeto com as keys das colunas
  const rows = [
    { cod: 'MP-01001', mat: 'Glicerina USP', pesado: '44983,21', status: 'OK' },
    { cod: 'MP-01002', mat: 'Fenoxietanol',  pesado: '3028,50',  status: 'Desvio' },
    { cod: 'MP-01003', mat: 'Ácido Cítrico', pesado: '12450,00', status: 'OK' }
  ];

  const tbl = document.getElementById('tbl');
  tbl.setAttribute('columns', JSON.stringify(columns));
  tbl.setAttribute('rows', JSON.stringify(rows));
</script>

Configuração de colunas

Cada objeto de coluna aceita as propriedades abaixo para controlar largura, alinhamento, tipografia mono e habilitar/desabilitar filtro e ordenação por coluna.

Propriedade Tipo Default Descrição
key string — (obrig.) Identificador da coluna. Deve bater com as keys dos objetos em rows.
label string "" Texto do cabeçalho da coluna (uppercase, peso 900).
width string (CSS) auto Largura da coluna. Aceita qualquer valor CSS ("120px", "18%", "10ch").
align "left" | "center" | "right" "left" Alinhamento do conteúdo (aplica em th e td).
mono boolean false Quando true, aplica fonte mono (códigos, números, IDs).
filterable boolean true Quando false, desabilita o filtro nesta coluna (sem ícone de funil).
sortable boolean true Quando false, desabilita a ordenação por essa coluna (header não vira clicável).
allowExport boolean true Quando false, a coluna é removida do CSV e do PDF. Não afeta a exibição na tabela.
cellStyle string (CSS) — CSS extra aplicado em cada td da coluna.

Renderização customizada (onrender)

O atributo onrender recebe código JS executado por célula. As variáveis disponíveis são cell (valor cru), col (objeto da coluna), row (objeto da linha) e rowIndex (índice global). O retorno é tratado como HTML — útil para badges, botões, ícones. 

tbl.setAttribute('onrender', `
  if (col.key === 'status') {
    var bg = row.status === 'OK'      ? '#1C7A38'
           : row.status === 'Desvio'  ? '#9A5A00'
           : row.status === 'Aprov.'  ? '#1A4A8C'
           : '#6B6B6B';
    return '<span style="display:inline-block;font-size:9px;font-weight:900;'
         + 'text-transform:uppercase;padding:3px 8px;border-radius:4px;'
         + 'background:' + bg + ';color:#fff">' + cell + '</span>';
  }
  return cell;  // demais colunas: valor cru
`);

Paginação

Ative com isenablepagination="true" e ajuste o tamanho de página com pagesize (default 25). O footer mostra controles Anterior / Próxima e o range visível. Filtros e ordenação são aplicados antes da paginação. 

<granado-table
    isenablepagination="true"
    pagesize="5"></granado-table>

Filtros e ordenação

Por padrão (isenablefilter="true" e isenablesort="true"), cada cabeçalho de coluna ganha um ícone de funil (popup com input de busca, substring case-insensitive) e fica clicável para ciclar a ordenação (nada → asc → desc → nada). Há também um filtro global à direita do tabletitle. Para desabilitar tudo, passe "false" nos atributos. Para desligar por coluna, use filterable: false / sortable: false no objeto coluna. 

<!-- Padrão: filtros e ordenação ativos -->
<granado-table tabletitle="Pesagens"></granado-table>

<!-- Desabilitar filtros/ordenação globalmente -->
<granado-table isenablefilter="false" isenablesort="false"></granado-table>

// Desabilitar por coluna (ex.: coluna de ações)
{ key: 'acao', label: 'Ação', filterable: false, sortable: false }

Eventos de clique

onclickevent dispara em single-click; ondoubleclickevent em duplo-clique. Variáveis disponíveis nos dois: row, rowIndex (índice global na lista original — não o índice paginado) e event. Quando ambos coexistem, o single-click é adiado ~250ms para não disparar junto com o duplo. 

tbl.setAttribute('onclickevent',
  "console.log('selecionou', row.cod);");

tbl.setAttribute('ondoubleclickevent',
  "alert('Abrir detalhes de ' + row.cod);");

Exportação CSV / PDF

Os botões Exportar CSV e Exportar PDF aparecem por padrão acima da tabela. CSV inclui BOM UTF-8 (abre certo no Excel). PDF abre uma janela com a tabela formatada e dispara o diálogo de impressão (usuário escolhe Salvar como PDF). A exportação sempre usa todas as linhas (ignora paginação). Células com HTML/CSS inline (ex.: um <span> estilizado) são exportadas apenas com o texto (as tags/estilos são removidos). Para remover uma coluna do CSV/PDF, use allowExport: false no objeto da coluna. Use exportfilename para fixar o nome base do arquivo. 

<granado-table
    tabletitle="Pesagens · turno A"
    exportfilename="pesagens_turno_A"></granado-table>

// Coluna que NÃO entra no CSV/PDF (ex.: coluna de ações):
{ key: 'acao', label: 'Ações', allowExport: false }

<!-- Para esconder os botões: -->
<granado-table
    isenableexportcsv="false"
    isenableexportpdf="false"></granado-table>

Visual customizado

color pinta a faixa lateral; tabletitle aparece como cabeçalho do card e do PDF; rowaltcolor controla a cor da zebra (default #FEF0CC); rowheight ajusta o padding vertical das células; tablelayout="fixed" força os widths estritos (conteúdo trunca em vez de empurrar a coluna). 

<granado-table
    color="#1A4A8C"
    tabletitle="Linha 03 · turno B"
    rowaltcolor="#E8F1FB"
    rowheight="10px"></granado-table>

<!-- Sem zebra -->
<granado-table rowaltcolor="none"></granado-table>

<!-- Layout fixo (widths estritos) -->
<granado-table tablelayout="fixed"></granado-table>

Linhas individuais podem sobrescrever a cor de fundo via uma chave especial _bg no objeto da linha — útil para destacar uma linha específica (ex.: a linha selecionada, ou um item em alerta). 

{ cod: 'MP-01006', mat: 'Óleo de Amêndoa', status: 'Desvio', _bg: '#FFE5E0' }

<granado-toast>

Notificação temporária com variantes success, error, warning e info. Não é colocado manualmente no HTML — usa-se via API estática GranadoToast.show(...), que cria o elemento, posiciona em um container fixo no canto da tela e remove automaticamente após duration ms (default 4000). Os containers (um por position) são criados sob demanda.

Import

<!-- Inclua o script no <head> (padrão Apriso) -->
<script src="[AprisoScripts]/WebComponents/granado-toast.js"></script>

Uso recomendado · API estática

A API estática é a forma idiomática de uso. GranadoToast.show(msg) aceita uma string simples; GranadoToast.show({...}) aceita opções completas. Os atalhos .success(), .error(), .warning() e .info() já injetam o variant. 

GranadoToast.success('Pesagem confirmada');
GranadoToast.error('Falha na balança', { heading: 'Erro' });
GranadoToast.warning('Lote em desvio', { heading: 'Atenção' });
GranadoToast.info('Aguardando supervisor');

Variantes

Quatro variantes determinam a cor da faixa lateral, do ícone e do título: success (verde), error (vermelho), warning (ouro) e info (azul, default). A escolha implica também o ícone padrão da notificação.

Heading e message

heading é o texto destacado (título); message é o corpo. Use só message para notificações curtas, ou os dois para alertas mais detalhados. title é aceito como sinônimo de heading por ergonomia. 

// Só message (mais comum)
GranadoToast.success('Salvo!');

// Heading + message
GranadoToast.show({
    variant: 'error',
    heading: 'Falha de comunicação',
    message: 'A balança B-01 não está respondendo. Verifique o cabo de rede.'
});

Duração

O atributo duration (em milissegundos, default 4000) controla quanto tempo o toast permanece. Use duration: 0 para tornar o toast permanente — fica até o usuário clicar no X ou até você chamar .dismiss() programaticamente. 

GranadoToast.show({ variant: 'info', message: 'Some em 2s', duration: 2000 });

// Permanente (até dismiss manual)
const t = GranadoToast.show({ variant: 'info', message: 'Conectando…', duration: 0 });

// Mais tarde, fechar via .dismiss()
t.dismiss();

Posição na tela

Seis posições disponíveis nos cantos/centros da tela. O componente cria um container por posição, então toasts em posições diferentes coexistem. Em top-* o mais novo aparece no topo da pilha (empurra os antigos para baixo); em bottom-* o mais novo fica mais perto da borda inferior. 

GranadoToast.show({
    variant: 'success',
    message: 'Toast no rodapé',
    position: 'bottom-center'
});

Fechar todos

GranadoToast.dismissAll() chama .dismiss() em todos os toasts ativos, em todas as posições. Útil para limpar a tela ao trocar de rota ou contexto. 

GranadoToast.dismissAll();

<granado-stepper>

Stepper horizontal estilo "cockpit de pesagem": bolinhas conectadas por uma linha, com título dentro e subtítulo embaixo. Cada step pode ser preenchido (concluído), atual (a posição em que o usuário está) ou apenas desabilitado. Setup totalmente data-driven via uma lista de objetos passada na propriedade .steps.

Cores são controladas em três grupos por estado — active (clicável), desactive (não-clicável) e current (atual) — para bolinhas, subtítulos e linha conectora. Cada step pode também sobrepor cor individualmente via iconColor/textColor.

Import

<!-- Inclua o script no <head> (padrão Apriso) -->
<script src="[AprisoScripts]/WebComponents/granado-stepper.js"></script>

Uso básico

Coloque um <granado-stepper> com id no HTML e atribua a propriedade .steps via JS. Cada item tem key + (opcionais) title, subtitle, isFilled, isCurrent, onClickEvent. 

<granado-stepper id="stp"></granado-stepper>

<script>
  document.getElementById('stp').steps = [
    { key: 'mp',   title: '✓', subtitle: 'Selecionar MP', isFilled: true,
      onClickEvent: "console.log('voltar', key)" },
    { key: 'ini',  title: '✓', subtitle: 'Iniciar',       isFilled: true,
      onClickEvent: "console.log('voltar', key)" },
    { key: 'bal',  title: '3', subtitle: 'Balança',        isFilled: true,
      isCurrent: true, onClickEvent: "alert('atual')" },
    { key: 'pes',  title: '4', subtitle: 'Pesagem',        isFilled: false },
    { key: 'conf', title: '5', subtitle: 'Confirmar',      isFilled: false }
  ];
</script>

Estados de um step

Três flags por step controlam o visual: isFilled pinta a bolinha (preenchido vs vazado), isCurrent marca o step atual (usa currentstepcolor) e a presença de onClickEvent define se o step é interativo. Steps não-clicáveis e não-current ficam apagados (opacity 0.45) — sinalizando "ainda não disponível". 

// 3 done (clicáveis), 1 atual, 1 pendente (apagado)

      onClickEvent: "console.log('a')" },
    { key: 'b', title: '✓', subtitle: 'Done · clicável', isFilled: true,
      onClickEvent: "console.log('b')" },
    { key: 'c', title: '3', subtitle: 'Atual',           isFilled: true,
      isCurrent: true, onClickEvent: "alert('atual')" },
    { key: 'd', title: '4', subtitle: 'Pendente · apagado', isFilled: false }
];

Paleta de cores

Em vez de pintar step a step, defina a paleta no nível do elemento. Os atributos activeiconcolor, desactiveiconcolor, activetextcolor, desactivetextcolor, activeconnectorcolor, desactiveconnectorcolor e currentstepcolor aplicam a todos os steps de uma só vez. 

<granado-stepper

// Atualizar a lista (cada set re-renderiza)

Ordem de aplicação das cores da bolinha

Quando o componente decide qual cor pintar a bolinha de um step, segue esta prioridade (do mais específico ao mais genérico):

  1. step.iconColor (override per-step) — vence sempre.
  2. currentstepcolor — se step.isCurrent === true.
  3. activeiconcolor — se o step é clicável (tem onClickEvent).
  4. desactiveiconcolor — caso contrário.

A mesma lógica vale para textColor (cor do subtitle), trocando iconcolor pelos respectivos textcolor.

Notas

  • Hover scale + padding: ao passar o mouse sobre uma bolinha clicável, aplica-se transform: scale(1.08) + sombra. O host tem padding-top/bottom: 6px para dar respiro vertical à animação (necessário porque overflow-x:auto força o overflow-y a clipar).
  • Full opacity em current: mesmo sem onClickEvent, um step com isCurrent: true não fica apagado — a opacidade reduzida sinalizaria desativado, mas o "atual" é o foco principal do stepper.
  • Linha conectora: a cor da linha entre dois steps depende apenas do step da esquerda — se ele é isFilled, a linha usa activeconnectorcolor; senão, desactiveconnectorcolor.

  • Items inline vs dinâmicos: para listas fixas, prefira items='[...]' direto no atributo — não precisa de JS. Para listas que vêm do servidor, use setAttribute('items', JSON.stringify(...)).
  • Lazy-props: o connectedCallback detecta atribuições feitas em .items ou .value antes do upgrade do custom element e re-aplica via setter da classe. Útil no Apriso, onde o script da página pode rodar antes do granado-submenu-pesagem.js.
  • change vs itemclick: itemclick dispara em todo click (mesmo no item já ativo). change só dispara quando o value realmente muda — use esse pra ações de navegação.
  • Card de contexto: o fundo é uma versão clarinha (88% misturado com branco) da contextcolor, então basta trocar uma cor pra adaptar a paleta inteira do chip.

Overlay de loading fullscreen que bloqueia toda a interação da tela enquanto exibe um ícone animado (escolhido via icon) e um texto dinâmico (default: "Aguarde..."). Casamento com requisições

O caso típico é envolver uma chamada assíncrona em try/finally para garantir que o overlay sempre feche, mesmo em caso de erro. 

async function salvarPesagem(payload) {
    GranadoPreloader.show('Salvando pesagem...');
    try {
        const resp = await fetch('/api/pesagem', { method: 'POST', body: JSON.stringify(payload) });
        if (!resp.ok) throw new Error('Falha');
        GranadoToast.success('Pesagem salva');
    } catch (e) {
        GranadoToast.error('Falha ao salvar', { heading: 'Erro' });
    } finally {
        GranadoPreloader.hide();
    }
}

Uso no Apriso (passo a passo)

Nas telas do Apriso onde você for usar chamadas async de DFC, o fluxo recomendado combina granado-preloader (bloqueia a tela + feedback visual) com granado-base-api (encapsula o try/catch e padroniza o objeto de resposta). Em 3 passos:

1° — Importar os dois scripts

No <head> da tela (ou no display global do Apriso), importe granado-preloader.js e granado-base-api.js: 

<script src="[AprisoScripts]/WebComponents/granado-preloader.js"></script>
<script src="[AprisoScripts]/WebComponents/granado-base-api.js"></script>

2° — Adicionar a DFC no menu "Advanced" do display

No FlexNet Studio, abra o display em que a chamada vai rodar e, no menu Advanced → DFCs, adicione o DFC que será chamado (ex.: GRD_API_SyncJDEOrder). Sem esse cadastro o Apriso não expõe o DFC para o context.callDFC(...) da página, e a chamada falha com erro de DFC não encontrado.

3° — Exemplo de uso

A função callDFCAsync(context, dfc, inputs, revisao?) exposta pelo granado-base-api.js já encapsula o try/catch — você não precisa mais tratar exceções manualmente. Basta mostrar o preloader antes da chamada, validar a resposta padronizada e esconder o preloader no fim: 

async function callMyDFCAsync() {
    _context = window.AprisoContext;
    GranadoPreloader.show('Sincronizando Ordem, aguarde...');

    await new Promise(e => setTimeout(() => e(), 1000)); // APENAS DEMONSTRACAO

    var response = await callDFCAsync(
        _context,
        'GRD_API_SyncJDEOrder',
        { orderNumber: '1000000002' }
    );

    if (response.responseResultData == null || !response.responseResultData.success) {
        console.log("Erro ao executar DFC xx");
        GranadoPreloader.hide();
        return;
    }

    console.log(response);
    GranadoPreloader.hide();
}

Formato da resposta

callDFCAsync retorna sempre um objeto com a mesma forma — tanto no sucesso quanto quando a DFC dispara erro. Não há exceção propagada para o chamador; basta inspecionar responseHTTPStatus e responseResultData: 

{
    responseHTTPStatus: 200,           // STATUS HTTP
    responseHTTPMessage: 'OK',          // MESSAGE HTTP
    responseResultData: result         // OBJETO DE RETORNO DA DFC
}
Propriedade Tipo Descrição
responseHTTPStatus number Status HTTP. 200 no sucesso; em erro, usa ex.status/ex.httpStatus da exceção do Apriso ou cai para 500.
responseHTTPMessage string Mensagem HTTP. "OK" no sucesso; em erro, vem de ex.message da exceção (Apriso Literal Errors trazem informação adicional em ex.literalId).
responseResultData object | null Objeto retornado pela DFC (External Outputs) no sucesso; null quando a chamada cai no catch. Sempre cheque por null antes de acessar response.responseResultData.xxx.

DFC em múltiplas etapas: entre awaits sequenciais, use GranadoPreloader.setText() para informar ao operador em qual passo está (validar → gravar → confirmar): 

GranadoPreloader.show('Validando dados...');
var r1 = await callDFCAsync(_context, 'GRD_API_Validar',    inputs);
if (r1.responseResultData == null) { GranadoPreloader.hide(); return; }

GranadoPreloader.setText('Gravando pesagem...');
var r2 = await callDFCAsync(_context, 'GRD_API_Gravar',     inputs);
if (r2.responseResultData == null) { GranadoPreloader.hide(); return; }

GranadoPreloader.setText('Confirmando...');
var r3 = await callDFCAsync(_context, 'GRD_API_Confirmar', inputs);

GranadoPreloader.hide();

Mesma ideia do granado-message-popup (callout no estilo abox, com tipos success/error/warning/info), porém com vários botões de ação (1 a 3) — cada botão dispara o seu próprio clique.

Uso

Defina os botões em buttons (1 a 3). Cada item aceita text, value, variant (primary/ghost) e um onClick. Todo clique também emite o evento option-click.

  el.addEventListener('option-click', function (e) { console.log(e.detail); });
</script>

Como definir os botões

A lista de botões pode ser passada de duas formas. A diferença está só no onClick: como atributo ele é texto (string); pela propriedade JS ele pode ser uma função.

1) Pelo atributo buttons (JSON)

Você escreve a lista direto no HTML, como um texto JSON. Aqui o onClick é uma string de código (executada quando o botão é clicado). 

<granado-option-popup type="warning" title="Reativar sala?"
  message="Deseja reativar a sala?"
  buttons='[
    { "text": "Cancelar", "value": "cancel", "variant": "ghost" },
    { "text": "Reativar", "value": "reativar", "onClick": "reativarSala()" }
  ]'></granado-option-popup>

2) Pela propriedade JS el.buttons (array) — recomendado

Você pega o elemento no JavaScript e atribui um array. Aqui o onClick pode ser uma função de verdade — sem precisar virar texto — o que é mais simples de ler e dá acesso direto ao detail do clique. 

var pop = document.getElementById('pop');

pop.buttons = [
  { text: 'Cancelar', value: 'cancel', variant: 'ghost' },
  { text: 'Reativar', value: 'reativar',
    onClick: function (detail) { reativarSala(detail.value); } }   // <- função
];
pop.open();

A API estática GranadoOptionPopup.show({ ..., buttons }) usa esse mesmo formato de array (com onClick como função). Em resumo: se você cria o popup por JavaScript, prefira el.buttons = [...] com funções; o atributo JSON é para quando a lista já vem fixa no HTML.

Ao informar limiteInferior e/ou limiteSuperior numa linha: os limites são anexados entre parênteses à característica, o campo Valor passa a aceitar apenas decimal e a coluna Check vira um Status automático e bloqueado (✓ OK dentro dos limites, ✕ Fora fora — no retorno continua true/false). 

<script>

    title: '🌡️ Condições Ambientais da Sala',
    checklistId: '100000003',
    checklistCode: 'CHECKLIST_AMBIENTAL',
    headerInformation: 'Itens fora do limite bloqueiam o início das ordens.',
    data: [
      // ambos os limites -> característica vira "Temperatura °C (18 – 24)"
      { id: 'temp', caracteristica: 'Temperatura °C', valor: '21,3', limiteInferior: 18, limiteSuperior: 24 },
      // só superior -> "Umidade Relativa % (≤ 60)"
      { id: 'ur',   caracteristica: 'Umidade Relativa %', valor: '58', limiteSuperior: 60 },
      // só inferior -> "Pressão Diferencial Pa (≥ 5)" · 3,2 < 5 => Status "✕ Fora"
      { id: 'pd',   caracteristica: 'Pressão Diferencial Pa', valor: '3,2', limiteInferior: 5 }
    ],
    onConfirm: function (d) { console.log(d.checklistId, d.checklistCode, d.rows); }   // status: true (OK) | false (Fora), automático
  });
</script>

<granado-gallery>

Galeria vertical de cards — mesma ideia da versão mobile da fila em "Pesagem › Seleção de Ordem de Pesagem". Cada card mostra (todos opcionais) title, subtitle, data e um badge de status; ao clicar, devolve o item inteiro + metadata.

Import

<!-- Inclua o script no <head> (padrão Apriso) -->
<script src="[AprisoScripts]/WebComponents/granado-gallery.js"></script>

Uso

Passe os itens pela propriedade JS .data (array). O clique dispara o evento item-click e chama .onItemClick(detail) se definido. 

<granado-gallery id="g"></granado-gallery>
<script>
  var g = document.getElementById('g');
  g.data = [
    { title: 'OP-2026-0416', subtitle: 'Loção Rosa 200ml', data: '18/05/2026',
      status: 'NOVO', metadata: { op: 'OP-2026-0416', sala: 'A' } },
    { title: 'OP-2026-0414', subtitle: 'Creme 150g', data: '17/05/2026', status: 'PESANDO' }
  ];
  g.onItemClick = function (d) { console.log(d.title, d.status, d.metadata); };
</script>

Sem JS — tudo por atributos

Também dá para configurar 100% no HTML: os itens vão no atributo data como JSON e o clique em onitemclick (string JS que recebe event e detail).

    onitemclick="console.log(detail.title, detail.status, detail.metadata)"
    data='[
      {"title":"OP-2026-0416","subtitle":"Loção Rosa 200ml","data":"18/05/2026","status":"NOVO","metadata":{"op":"OP-2026-0416","sala":"A"}},
      {"title":"OP-2026-0414","subtitle":"Creme 150g","data":"17/05/2026","status":"PESANDO"},
      {"title":"BAL-021","status":"FORA"}

Obs.: no atributo data use JSON válido (chaves e strings com aspas duplas) e envolva o valor com aspas simples. metadata aceita qualquer objeto e volta no detail do clique.
