<granado-chart-bar>

Gráfico de barras em SVG (Light DOM, estilos inline). Suporta barras simples, agrupadas, empilhadas, empilhadas 100% e target (faixa mín→máx), na vertical ou horizontal, com legenda, grade, tooltip, rótulos de valor, animação, meta global e exportação PNG/CSV. Cores padrão da paleta Granado (validada p/ legibilidade e daltonismo) e layout responsivo.

Import

<!-- Inclua o script no <head> (padrão Apriso) -->
<script src="[AprisoScripts]/WebComponents/granado-chart-bar.js"></script>

Uso — via script (recomendado)

Defina xField, yFields e data por propriedade (aceita objetos/funções).

<granado-chart-bar id="c" title="Vendas por mês" show-values="true"></granado-chart-bar>
<script>
  var c = document.getElementById('c');
  c.xField  = 'month';
  c.yFields = [{ field: 'sales', label: 'Vendas' }];
  c.data    = [
    { month: 'Jan', sales: 120 }, { month: 'Fev', sales: 180 },
    { month: 'Mar', sales: 150 }, { month: 'Abr', sales: 220 }
  ];
</script>

Uso — declarativo (HTML)

data e y-fields também aceitam JSON por atributo (aspas simples). Útil quando nenhuma série usa funções (formatter/tooltipFormatter). 

<granado-chart-bar type="grouped" title="Vendas x Meta"
  x-field="month"
  y-fields='[{"field":"sales","label":"Vendas"},{"field":"target","label":"Meta"}]'
  data='[{"month":"Jan","sales":120,"target":150},
         {"month":"Fev","sales":180,"target":160}]'></granado-chart-bar>

Tipos — type

O type define o layout das séries; as séries vêm das yFields (formato largo) ou do x2Field (formato longo). grouped com 1 série parece simple (não há o que agrupar).

typeO que faz
simpleUma barra por categoria (usa a 1ª série).
groupedSéries lado a lado dentro de cada categoria.
stackedSéries empilhadas (soma por categoria).
stacked100Empilhadas normalizadas a 100% (participação).
targetBarra flutuante de faixa: usa 2 campos [min, max].
 
// target usa 2 campos: [min, max]
c.type    = 'target';
c.xField  = 'employee';
c.yFields = [{ field: 'min' }, { field: 'max', label: 'Faixa' }];
c.data    = [
  { employee: 'João',  min: 4000, max: 7000 },
  { employee: 'Maria', min: 5000, max: 8500 }
];

Orientação — orientation

vertical (default) ou horizontal.

2º agrupador — x2Field

xField é a dimensão do eixo; x2Field é o segundo agrupador (a dimensão das séries/cor). Use para dados em formato longo — uma linha por combinação (xField, x2Field, valor). O valor plotado é yFields[0].field. Equivale a passar várias yFields (formato largo). 

c.xField  = 'quarter';
c.x2Field = 'region';            // uma série por região
c.yFields = [{ field: 'sales' }];  // valor do pivot
c.data    = [
  { quarter: 'Q1', region: 'Sul',   sales: 120 },
  { quarter: 'Q1', region: 'Norte', sales: 90  },
  { quarter: 'Q2', region: 'Sul',   sales: 150 }
];

Meta global — overallTarget

Meta única que vale para todos os registros, desenhada como linha de referência atravessando o gráfico (horizontal nas barras verticais; vertical nas horizontais). A meta entra no domínio do eixo, então a linha sempre aparece. 

// número simples
c.overallTarget = 200;

// objeto
c.overallTarget = { value: 200, label: 'Meta 200',
                    color: '#8C1A1A', dashed: true, showValue: true };

// várias linhas
c.overallTarget = [{ value: 200, label: 'Meta' },
                   { value: 260, color: '#1A4A8C' }];

Formato da barra — barRole

Definido por série (em yFields):

barRoleRenderização
defaultBarra cheia (padrão).
dashedContorno tracejado, preenchimento fraco (ex.: limite/referência).
lineMarcador de linha na ponta do valor (ex.: meta/alvo).
 
c.yFields = [
  { field: 'sales',  label: 'Vendas' },                        // default
  { field: 'target', label: 'Meta',   barRole: 'line'   },
  { field: 'limite', label: 'Limite', barRole: 'dashed' }
];

Exportação

Por padrão exibe ⬇ PNG (imagem, com título + legenda) e ⬇ CSV (dados em tabela). Oculte com show-export="false"; posicione com export-position. Também há métodos JS. 

// métodos
c.exportPNG('vendas');      // ou c.exportPNG('vendas', 3) p/ 3x
c.exportCSV('vendas');

// posição dos botões
<granado-chart-bar export-position="bottom-left" ...>
// top-right (default) | top-left | bottom-right | bottom-left

Clique & seleção — bar-click

Cada barra é clicável: ela fica destacada e as demais esmaecem; clicar de novo na mesma barra volta tudo ao normal (toggle). O evento bar-click devolve os valores da barra e de toda a categoria. Clique nas barras abaixo:

— clique numa barra —
c.addEventListener('bar-click', function (ev) {
  var d = ev.detail;
  // { index, category, series, field, value, color,
  //   categoryValues, selected }  (min/max em 'target')
  console.log(d.category, d.series, d.value, d.categoryValues);
  if (d.selected) { /* barra ficou destacada */ }
});

Evento & métodos

.exportPNG(nome?, escala?)Baixa a imagem .png (escala default 2). Inclui título + legenda.
.exportCSV(nome?)Baixa os dados como tabela .csv (separador ; + BOM UTF-8).

Notas

  • type × séries: type só define o layout. As séries vêm das yFields (largo) ou do x2Field (longo).
  • Soma: categorias repetidas são sempre somadas (largo, longo e target).
  • Paleta Granado: cores em ordem fixa (verde · azul · ouro · vermelho · violeta · ouro-claro); passe color por série para sobrepor.
  • Light DOM + inline: sem Shadow DOM e sem stylesheet injetado; SVG self-contained (export sem recursos externos).

<granado-chart-line>

Gráfico de linhas em SVG — mesma ideia do granado-chart-bar. Suporta linha, área, área empilhada e área 100%, com legenda, grade, tooltip, pontos, curva (smooth), meta global, seleção no clique e exportação PNG/CSV. Só na vertical.

Import

<script src="[AprisoScripts]/WebComponents/granado-chart-line.js"></script>

Uso — via script

<granado-chart-line id="c" title="Vendas por mês" show-values="true"></granado-chart-line>
<script>
  var c = document.getElementById('c');
  c.xField  = 'month';
  c.yFields = [{ field: 'sales', label: 'Vendas' }];
  c.data    = [
    { month: 'Jan', sales: 120 }, { month: 'Fev', sales: 180 },
    { month: 'Mar', sales: 150 }, { month: 'Abr', sales: 220 }
  ];
</script>

Tipos — type

typeO que faz
lineLinhas (default).
areaLinha com área preenchida.
stackedÁreas empilhadas (soma por categoria).
stacked100Áreas empilhadas normalizadas a 100%.

Curva, tracejado, pontos — props de série

c.yFields = [
  { field: 'sales',  label: 'Vendas', smooth: true },              // curva
  { field: 'target', label: 'Meta',   smooth: true, dashed: true, // tracejada
    showDots: false }                                     // sem pontos
];

Clique & seleção — point-click

Clicar num ponto destaca a série e esmaece as demais (toggle); o evento devolve os valores.

c.addEventListener('point-click', function (ev) {
  // { index, category, series, field, value, color,
  //   categoryValues, selected }
  console.log(ev.detail);
});

API

Mesmos atributos do granado-chart-bar (data, x-field, y-fields, x2-field, overall-target, title, show-legend/grid/values/tooltip, animated, show-export, export-position, height, width), exceto: não tem orientation nem type="target". O type é line/area/stacked/stacked100.

Campos de série (yFields[i]) específicos de linha

CampoTipoDescrição
lineWidthnumberEspessura da linha (default 2.5).
dashedbooleanLinha tracejada.
showDotsbooleanPontos nos vértices (default true).
smoothbooleanCurva suave (spline) em vez de retas.
areabooleanPreenche a área desta série (mesmo em type "line").
color · label · formatter · tooltipFormatter · sort · opacity—Iguais ao granado-chart-bar.

Evento & métodos

NomeDescrição
point-clickdetail: { index, category, series, field, value, color, categoryValues, selected }.
.exportPNG(nome?, escala?)Imagem .png (título + legenda).
.exportCSV(nome?)Dados em tabela .csv (; + BOM).

<granado-chart-pie>

Gráfico de pizza em SVG — mesma ideia dos demais. Suporta pizza, rosca (donut) e semicírculo (gauge), com legenda, tooltip, rótulos de %, rótulo central, seleção no clique e exportação PNG/CSV.

Import

<script src="[AprisoScripts]/WebComponents/granado-chart-pie.js"></script>

Uso — via script

A pizza tem um valor por rótulo: use x-field (rótulo) + value-field (valor).

<granado-chart-pie id="c" title="Pesagens por sala" show-values="true"></granado-chart-pie>
<script>
  var c = document.getElementById('c');
  c.xField     = 'sala';
  c.valueField = 'total';
  c.data = [
    { sala: 'Sala A', total: 120 }, { sala: 'Sala B', total: 80 },
    { sala: 'Sala C', total: 60 }, { sala: 'Sala D', total: 40 }
  ];
</script>

Tipos — type

typeO que faz
piePizza cheia (default).
donutRosca (raio interno). Aceita center-label.
semiSemicírculo (gauge). Aceita center-label.

Cores por fatia + clique

Cada item de data pode ter color. Clicar numa fatia destaca (toggle) e emite slice-click. A legenda também é clicável.

c.xField = 'status'; c.valueField = 'qtd';
c.data = [
  { status: 'OK',     qtd: 320, color: '#1F7A3D' },
  { status: 'Alerta', qtd: 60,  color: '#9A7520' },
  { status: 'Fora',   qtd: 40,  color: '#B5342F' }
];
c.tooltipFormatter = function (ctx) {
  return '<b>' + ctx.label + '</b><br>' + ctx.value + ' un · ' + ctx.percent.toFixed(0) + '%';
};
c.addEventListener('slice-click', function (ev) {
  console.log(ev.detail);  // { index, label, value, percent, color, selected }
});

NomeTipoDefaultDescrição

x-fieldstring—Chave do rótulo da fatia. Em JS .xField.

typestring"pie"pie · donut · semi.

.formatter / .tooltipFormatterfunção—Formata valor / HTML do tooltip (via JS).
